# Agent Instructions for gh_linkshortener

This document outlines the coding standards, conventions, and patterns for LLM agents working on this project. All new code should strictly adhere to these guidelines.

## Quick Reference

| Category        | Standard                      |
| --------------- | ----------------------------- |
| Language        | TypeScript 5+ (strict mode)   |
| Framework       | Next.js 16.2.0 (App Router)   |
| UI Library      | shadcn/ui + Tailwind CSS 4    |
| Database        | Drizzle ORM + Neon Serverless |
| Authentication  | Clerk (App Router)            |
| Runtime         | Node.js ES2017+               |
| Package Manager | npm                           |

## Core Principles

1. **Strict TypeScript** - Always use strict mode. No `any` types without justification.
2. **App Router Only** - Never use Pages Router (`pages/` dir). Use `app/` directory exclusively.
3. **Client Components Explicit** - Always add `'use client'` at the top of client components.
4. **Zero Authentication Shortcuts** - Follow Clerk best practices. Never hardcode auth logic.
5. **Responsive by Default** - All layouts must be responsive. Use Tailwind breakpoints.

## Naming Conventions

- **Files**: `kebab-case` for utilities/helpers, `PascalCase` for React components
- **Functions/Variables**: `camelCase`
- **Types/Interfaces**: `PascalCase`
- **Constants**: `UPPER_SNAKE_CASE`
- **Props Interface**: `ComponentNameProps`
- **CSS Classes**: Tailwind utilities + kebab-case custom classes

## Import Organization

```typescript
// 1. External packages
import { useRouter } from "next/navigation";
import { ClerkProvider } from "@clerk/nextjs";

// 2. Internal modules (from @/)
import { Button } from "@/components/ui/button";
import { db } from "@/lib/db";

// 3. Relative imports (if necessary)
import { helper } from "./helper";

// Blank line between groups
```

## TypeScript Configuration

All files must be `.ts` or `.tsx`. Key compiler options enforced:

- `strict: true` - All strict checks enabled
- `noImplicitAny: true` - No untyped variables
- `jsx: react-jsx` - React 17+ JSX transform
- `moduleResolution: bundler` - Modern module resolution

**Never ignore TypeScript errors.** If a type is complex, define it explicitly.

## Authentication with Clerk

Always:

- ✅ Use `@clerk/nextjs` package
- ✅ Add `<ClerkProvider>` in root layout inside `<body>`
- ✅ Use `<Show when="signed-in">` / `<Show when="signed-out">`
- ✅ Protect routes using Clerk's built-in protection
- ✅ Use `auth()` from `@clerk/nextjs/server` in Server Components

Never:

- ❌ Use deprecated `<SignedIn>` / `<SignedOut>` components
- ❌ Use old `authMiddleware()` - use `clerkMiddleware()`
- ❌ Import from inconsistent paths - always use `@clerk/nextjs` or `@clerk/nextjs/server`

## Database Patterns

- Schemas live in `lib/db/schema.ts`
- Queries go in `lib/db/queries.ts` or feature-specific files
- Always use prepared statements (Drizzle's built-in)
- Never write raw SQL without escaping and review
- Migrations are auto-tracked by drizzle-kit

## Component Best Practices

### Server Components (Default)

```typescript
// app/dashboard/page.tsx
export default async function Dashboard() {
  const data = await fetchData() // Database queries here
  return <div>{/* render with data */}</div>
}
```

### Client Components (Explicit)

```typescript
'use client'
import { useState } from 'react'

export function Counter() {
  const [count, setCount] = useState(0)
  return <button onClick={() => setCount(count + 1)}>{count}</button>
}
```

## Code Quality Standards

- **ESLint**: Run `npm run lint` before committing
- **No unused variables** - Remove dead code
- **Meaningful names** - Self-documenting code
- **Comments only for "why"** - Code should explain "what"
- **DRY principle** - Extract reusable logic to utilities or components

## Performance Guidelines

- Lazy load heavy components: `const Modal = dynamic(() => import('./Modal'))`
- Use `<Image>` from `next/image` with explicit `width` and `height`
- Optimize database queries - use Drizzle's `.limit()`, `.select()`
- Avoid `useEffect` with missing dependencies
- Memoize expensive computations: `useMemo`, `useCallback`

## Environment Variables

Use `.env.local` (NOT committed). For shared config:

- Public vars: `NEXT_PUBLIC_*`
- Secret vars: `CLERK_SECRET_KEY`, database URLs, etc.

Never commit `.env.local` or hardcode secrets.

## Troubleshooting

**"Cannot find module"** - Check `tsconfig.json` paths. Use `@/` for project imports.

**"Property does not exist"** - Likely missing await on async function. Check Server vs Client context.

**"Clerk is not initialized"** - Ensure `<ClerkProvider>` wraps your app in layout.tsx inside `<body>`.

**Type errors in database** - Update Drizzle with `npm install drizzle-orm drizzle-kit`.
